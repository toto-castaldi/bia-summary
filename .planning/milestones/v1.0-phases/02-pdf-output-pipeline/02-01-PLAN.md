   a) `parseInputFilename(filename: string): ParsedFilename | null`
      - Regex pattern: `/^(\d{2})_(\d{2})_(\d{4})\s*-\s*(.+?)\s*-\s*Report di stampa/i`
      - Extracts DD, MM, YYYY and client name from Bodygram filename pattern
      - Returns `{ clientName: trimmed_name, date: "YYYY_MM_DD" }` or null on no match
      - Per D-02: this is the PRIMARY path

   b) `extractFromMarkdown(markdown: string): ParsedFilename | null`
      - Per D-02: this is the FALLBACK when filename parsing fails
      - Extract name via regex: `/\*?\*?Nome\*?\*?:\s*(.+)/i`
      - Extract date via regex: `/\*?\*?Data\s*(?:esame|visita)?\*?\*?:\s*(\d{1,2})[\/\-\.](\d{1,2})[\/\-\.](\d{4})/i`
      - Returns ParsedFilename with extracted values, null if name not found
      - If date not found but name found, use current date as `YYYY_MM_DD`

   c) `generateOutputFilename(parsed: ParsedFilename): string`
      - Returns `${parsed.date} - ${parsed.clientName} - Riassunto BIA.pdf` (per D-03)

   d) `generateOutputPath(inputPath: string, markdown: string, customOutputPath?: string): string`
      - If `customOutputPath` provided, return `path.resolve(customOutputPath)` (per CLI-04)
      - Otherwise: try `parseInputFilename(path.basename(inputPath))`
      - If that returns null: try `extractFromMarkdown(markdown)` (per D-02 fallback)
      - If both return null: use current date and generic name `YYYY_MM_DD - Riassunto BIA.pdf`
      - Output file is placed in the same directory as the input PDF (per D-03 convention)

   Imports needed: `path` from `node:path`, `ParsedFilename` from `./types.js`

  <action>
Create `src/clients/cloudconvert.ts` with a single exported function:

```typescript
export async function convertMarkdownToPdf(
  markdown: string,
  apiKey: string,
): Promise<Buffer>
```

Implementation steps:
1. Import `CloudConvert` from `"cloudconvert"` (default export)
2. Instantiate: `new CloudConvert(apiKey)` -- pass key explicitly, SDK does NOT auto-read env vars (per Pitfall 5)
3. Create a job with three tasks (per Pattern 1 -- import/raw, not import/upload):
   - `"import-md"`: operation `"import/raw"`, file: `markdown` (the string content), filename: `"summary.md"`
   - `"convert-to-pdf"`: operation `"convert"`, input: `["import-md"]`, output_format: `"pdf"`
     (Per D-01: start with md-to-pdf / Pandoc engine. Do NOT specify engine explicitly -- let CloudConvert auto-select. Per D-04: no page_size param needed since Pandoc defaults to A4 for European locales.)
   - `"export-pdf"`: operation `"export/url"`, input: `["convert-to-pdf"]`
4. Wait for job: `await cloudConvert.jobs.wait(job.id)` -- SDK handles polling internally (per Anti-Pattern in RESEARCH.md: do NOT poll manually)
5. Get export URLs: `const exportUrls = cloudConvert.jobs.getExportUrls(completed)`
6. Validate: throw if `!exportUrls.length || !exportUrls[0].url`
7. Download immediately (per Pitfall 3 -- URLs expire): `const response = await fetch(exportUrls[0].url)`
8. Validate response: throw if `!response.ok` with HTTP status in message
9. Convert to Buffer: `Buffer.from(await response.arrayBuffer())`
10. Return the Buffer

Follow the same module pattern as `src/clients/claude.ts`: single function export, explicit parameters, descriptive error messages.
