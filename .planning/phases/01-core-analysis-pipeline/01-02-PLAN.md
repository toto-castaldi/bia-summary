2. Create `src/clients/claude.ts` with the following:

   **Import:** `Anthropic` from `"@anthropic-ai/sdk"`, `readFile` from `"node:fs/promises"`, `AnalysisResult` from `"../types.js"`

   **Export function `analyzePdf(pdfPath: string, promptContent: string): Promise<AnalysisResult>`:**
   - Read PDF file: `const pdfBuffer = await readFile(pdfPath)`
   - Base64 encode: `const pdfBase64 = pdfBuffer.toString("base64")`
   - Create Anthropic client: `const client = new Anthropic()` (reads ANTHROPIC_API_KEY from process.env automatically)
   - Call `client.messages.create()` with:
     - `model: "claude-sonnet-4-20250514"` (Claude's discretion, research recommends Sonnet 4)
     - `max_tokens: 8192` (Claude's discretion, minimum 4096 per D-10, 8192 for safety margin)
     - `messages`: single user message with content array:
       - First element: `{ type: "document", source: { type: "base64", media_type: "application/pdf", data: pdfBase64 } }` (PDF BEFORE text per Anthropic docs best practice)
       - Second element: `{ type: "text", text: promptContent }`
   - Extract text from response: find the content block where `block.type === "text"`, throw `Error("Claude returned no text response")` if not found
   - Return `AnalysisResult`:
     - `markdown`: the text block's `text` field
     - `stopReason`: `response.stop_reason`
     - `model`: `response.model`
     - `inputTokens`: `response.usage.input_tokens`
     - `outputTokens`: `response.usage.output_tokens`

   Note: This function does NOT validate the response (no math checks, no truncation warnings). That is the pipeline's responsibility. This function is a clean API wrapper.

3. Run `npx tsc --noEmit` to verify the file compiles.

1. Create `src/pipeline.ts` with the following:

   **Imports:**
   - `loadPrompt` from `"./config.js"`
   - `analyzePdf` from `"./clients/claude.js"`
   - `PipelineOptions`, `AnalysisResult` from `"./types.js"`
   - `ora` from `"ora"` (spinner for progress, must use `stream: process.stderr` per D-05)

   **Export function `runPipeline(options: PipelineOptions): Promise<AnalysisResult>`:**

   Step 1 -- Start spinner (to stderr):
   - `const spinner = ora({ text: "Loading prompt...", stream: process.stderr }).start()`

   Step 2 -- Load prompt (D-01):
   - Call `const prompt = await loadPrompt(options.inputPath)`
   - `spinner.text = "Analyzing BIA report with Claude..."`

   Step 3 -- Analyze PDF:
   - Call `const result = await analyzePdf(options.inputPath, prompt)`
   - `spinner.succeed("Analysis complete")`

   Step 4 -- Validate response (D-09, D-10):
   - Call `validateResponse(result)` (internal function, see below)

   Step 5 -- Return result

   **Internal function `validateResponse(result: AnalysisResult): void`:**

   Truncation check (D-10):
   - If `result.stopReason === "max_tokens"`, write to stderr: `"WARNING: Response was truncated (stop_reason: max_tokens). Summary may be incomplete."`

   Math validation (D-09):
   - Best-effort regex extraction from `result.markdown`:
     - Look for FM (fat mass) value in kg: pattern like `/(?:massa grassa|FM|fat mass)[^0-9]*?(\d+[.,]\d+)\s*kg/i`
     - Look for FFM (fat-free mass) value in kg: pattern like `/(?:massa magra|FFM|fat-free mass)[^0-9]*?(\d+[.,]\d+)\s*kg/i`
     - Look for body weight: pattern like `/(?:peso|weight)[^0-9]*?(\d+[.,]\d+)\s*kg/i`
   - If all three values extracted and `|FM + FFM - weight| > 1.0`, write to stderr: `"WARNING: Math check failed — FM ({fm} kg) + FFM ({ffm} kg) = {sum} kg, but body weight is {weight} kg (difference: {diff} kg)"`
   - Look for TBW, ECW, ICW similarly. If all three extracted and `|TBW - (ECW + ICW)| > 0.5`, write to stderr: `"WARNING: Math check failed — ECW ({ecw} kg) + ICW ({icw} kg) = {sum} kg, but TBW is {tbw} kg"`
   - If parsing fails for any value, skip that validation silently (best-effort per research open question 3)
   - All warnings go to stderr via `console.error()` (D-05)

2. Run `npx tsc --noEmit` to verify compilation.
