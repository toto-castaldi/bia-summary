2. **src/retry.ts** (NEW file) -- Create a shared withRetry utility for CloudConvert calls. Implementation:
   - Export `RetryOptions` interface with fields: `maxRetries` (default 3), `baseDelayMs` (default 1000), `maxDelayMs` (default 10000), `retryableStatuses` (default [429, 500, 502, 503]), `onRetry` optional callback `(attempt: number, delayMs: number, error: unknown) => void`.
   - Export `async function withRetry<T>(fn: () => Promise<T>, options?: Partial<RetryOptions>): Promise<T>`.
   - Loop from attempt 0 to maxRetries. On catch: if last attempt or not retryable, rethrow. Otherwise call onRetry callback, sleep, and retry.
   - `isRetryable(error, statuses)`: Check `error instanceof Error && error.cause instanceof Response` then `statuses.includes(error.cause.status)`. Also retry on `TypeError` (network errors).
   - `calculateDelay(attempt, baseMs, maxMs, error)`: First check `error.cause.headers.get("retry-after")` for Retry-After header (parse as integer seconds). Otherwise use exponential backoff with full jitter: `Math.random() * Math.min(baseMs * 2^attempt, maxMs)`.
   - Private `sleep(ms)` function using `new Promise(resolve => setTimeout(resolve, ms))`.
   - IMPORTANT: This utility is ONLY for CloudConvert. Do NOT use it for Anthropic SDK calls (the SDK has built-in retry -- see anti-pattern in RESEARCH.md).

3. **src/clients/claude.ts** -- Add `maxRetries: 3` to the Anthropic constructor (PDF-04). Change:

   const client = new Anthropic();

   ```typescript
   const client = new Anthropic({ maxRetries: 3 });
   ```
   This is the ONLY change needed. Do NOT wrap analyzePdf in withRetry -- the SDK handles retry internally (avoids double-retry pitfall from RESEARCH.md Pitfall 1).

   ```typescript
   export async function loadPrompt(
     pdfPath: string,
     templateVars?: TemplateVars,
   ): Promise<string> {
     // ... existing file loading logic stays the same ...
     let content = await readFile(promptPath, "utf-8");

     if (templateVars?.clientName) {
       content = content.replaceAll("{{CLIENT_NAME}}", templateVars.clientName);
     }
     if (templateVars?.examDate) {
       const [y, m, d] = templateVars.examDate.split("_");
       content = content.replaceAll("{{EXAM_DATE}}", `${d}/${m}/${y}`);
     }

     return content;
   }
   ```
   The `examDate` is in YYYY_MM_DD format from parseInputFilename; convert to DD/MM/YYYY for Italian convention. If templateVars is undefined or fields are undefined, skip replacement (variables remain as literal text -- harmless per CONTEXT.md).
